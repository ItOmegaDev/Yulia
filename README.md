# 🧵 Нитка — Екологічна Майстерня Індивідуального Пошиву

Ласкаво просимо до репозиторію **Нитка** — висококласної веб-платформи для екологічного пошиву одягу та предметів побуту за індивідуальними мірками клієнтів. 

Цей проєкт створено для автоматизації взаємодії між замовниками та професійними кравцями. Він поєднує в собі вишуканий мінімалістичний дизайн в етно-модерновому стилі та надійну full-stack архітектуру для стабільної щоденної роботи бізнесу.

---

## 🎨 Філософія та Візуальна Ідентичність

Майстерня **Нитка** сповідує принципи **повільної моди (slow fashion)** та свідомого споживання. Візуальний стиль платформи відображає ці цінності:
- **Колористика**: Ексклюзивна палітра «Cosmic Slate» та «Editorial Warm» — м'які кремові тони (`#F5F2EB`), глибокий графітовий відтінок льону (`#2c2925` / `bg-editorial-dark`), вишукані теракотові та вохристі акценти.
- **Типографіка**: Витончені класичні serif-шрифти для заголовків гармонійно поєднуються зі зручним sans-serif для навігації та моноширинним "JetBrains Mono" для технічних параметрів чи мірок.
- **Ергономіка**: Максимальна увага до деталей — плавні мікро-анімації інтерфейсу за допомогою `motion/react`, відсутність зайвого візуального шуму та адаптивний інтерфейс для мобільних пристроїв і десктопів.

## ⚡ Ключовий Функціонал

### 🛍️ Клієнтська Зона (Магазин)
1. **Інтерактивний каталог**: Зручне групування замовлень за категоріями (Одяг, Предмети побуту, Аксесуари) та гнучке сортування (за ціною, популярністю чи базовим порядком).
2. **Конструктор мірок**: При виборі виробу клієнт може вибрати як стандартний розмір (S, M, L, XL), так і ввести власні індивідуальні параметри (обхват грудей, талії, стегон, зріст). 
3. **Гнучкий кошик**: Розрахунок вартості в реальному часі, збереження обраних параметрів шиття та оптимізоване додавання товарів без зайвого очікування чи помилок зникнення.
4. **Миттєве оформлення замовлення**: Клієнт обирає спосіб доставки (Нова Пошта, Самовивіз, Кур'єр) та оплати (Накладений платіж чи Банківський переказ за реквізитами IBAN), заповнює контактні дані, після чого замовлення відправляється на обробку.

### 👑 Панель Адміністратора (Для Кравців)
1. **Сучасний Dashboard**: Інтерактивна аналітика продажів за допомогою графіків `recharts` та `d3`, статистика середнього чека та найпопулярніших товарів.
2. **Керування замовленнями**: Зміна статусів замовлень («Нове», «В роботі», «Шиється», «Надіслано», «Виконано»), перегляд точних мірок клієнта для кожного виробу.
3. **Керування каталогом**: Створення, редагування та видалення виробів. Налаштування доступних розмірів, матеріалів, терміну виготовлення та детального опису.
4. **Редактор налаштувань сайту**: Керування контактами, адресою, описом переваг майстерні та швидке ввімкнення/вимкнення платіжних методів у реальному часі.
5. **Конфігуратор хмари**: Можливість динамічного підключення бази даних Firebase Firestore безпосередньо з інтерфейсу.

## 🛠️ Технологічний Стек

Проєкт побудований на базі сучасних та перевірених технологій для забезпечення найвищої швидкодії, відмовостійкості та чистоти коду:

- **Frontend**:
  - **React 18** (функціональні компоненти, хуки, розділений стан).
  - **Vite** — надшвидкий збирач та середовище розробки.
  - **TypeScript** — строга типізація для мінімізації помилок під час виконання.
  - **Tailwind CSS** — утилітарний CSS-фреймворк для точного та сучасного дизайну.
  - **Motion (Framer Motion)** — плавні фізичні анімації переходів, модальних вікон та кошика.
  - **Lucide React** — уніфікований набір векторних іконок преміум-якості.
  - **Recharts & D3** — побудова інтерактивних діаграм фінансової аналітики та продажів.

- **Backend & База Даних**:
  - **Express** (Node.js) — серверний проксі-шар та API маршрути для безпеки та гнучкості.
  - **Firebase Firestore** — персистентна хмарна база даних реального часу для збереження товарів, замовлень та налаштувань.
  - **Гнучка гібридна архітектура**: Автоматичний резервний перехід на локальне сховище (`localStorage`) у разі відсутності хмарного з’єднання. База даних автоматично синхронізує та копіює локальні дані у хмару при першому успішному підключенні, забезпечуючи нульову втрату даних!

## 🚀 Встановлення та Запуск

Щоб запустити проєкт локально на вашому комп'ютері, виконайте наступні кроки:

### 1. Клонування репозиторію
```bash
git clone https://github.com/your-username/nytka-workshop.git
cd nytka-workshop
```

### 2. Встановлення залежностей
Переконайтеся, що на вашому комп'ютері встановлено Node.js (рекомендовано версію 18 або новішу).
```bash
npm install
```

### 3. Налаштування змінних оточення (опціонально)
Проєкт підтримує роботу в локальному автономному режимі з коробки. Якщо ви хочете підключити власну базу даних Firebase Firestore, створіть файл `.env` у корені проєкту та вкажіть відповідні ключі:
```env
# .env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

### 4. Запуск у режимі розробки
Запустіть локальний сервер розробки:
```bash
npm run dev
```
Після цього відкрийте сторінку [http://localhost:3000](http://localhost:3000) у вашому браузері.

### 5. Створення фінальної збірки для Production
Для оптимізації та компіляції проєкту перед розгортанням на хостинг виконайте:
```bash
npm run build
```
Скомпільовані файли для клієнта будуть збережені в папці `/dist`, а серверна частина буде скомпільована в `/dist/server.cjs`.

Запуск у Production режимі:
```bash
npm run start
```

## 🛡️ Безпека та Правила Доступу (Firestore)

Для захисту конфіденційності мірок та замовлень клієнтів у файлі `firestore.rules` налаштовано суворі правила безпеки:
- **Товари (`products`)** та **Налаштування (`settings`)**: Доступні для публічного читання всім відвідувачам магазину, але модифікація дозволена лише авторизованим адміністраторам.
- **Замовлення (`orders`)**: Кожен відвідувач може створити (створити запис) нове замовлення, але читати, оновлювати або видаляти базу замовлень може виключно авторизований адміністратор у панелі керування.
